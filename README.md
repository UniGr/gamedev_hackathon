# 🚀 Project: Universal (Idle Clicker)

**Universal** — это idle-кликер о выживании одинокого корабля в недружелюбной галактике. Ваша цель: собирать мусор, расширять отсеки и накопить достаточно мощности для финального гиперпрыжка.

---

## 🛠 Архитектурные принципы

Проект строится на трех столпах:

1.  **Feature-based Folders:** Все ресурсы конкретной фичи (скрипты, сцены, спрайты) лежат в одной папке.
2.  **Event Bus (Сигнальная шина):** Объекты не знают друг о друге. Они общаются через глобальный синглтон `GameEvents`.
3.  **Composition over Inheritance:** Мы собираем функционал из маленьких узлов-компонентов, а не плодим гигантские иерархии наследования.

---

## 📂 Структура проекта

```text
res://
├── assets/                 # Общие ассеты (звуки, шрифты, палитры)
├── core/                   # Глобальные системы (Autoloads)
│   ├── game_events.gd      # "Почта" для сигналов между модулями
│   └── resource_manager.gd # Хранение Металла и Энергии
├── data/                   # Файлы баланса (.tres)
│   └── room_stats/         # Параметры комнат (цена, добыча)
├── entities/               # Игровые объекты
│   ├── ship/               # Сетка корабля и логика постройки
│   ├── modules/            # Сцены отсеков (Resource/Combat)
│   └── debris/             # Космический мусор и спавнер
├── ui/                     # Интерфейс (HUD, Магазин, Меню)
└── shared/                 # Общие компоненты (Health, Clickable и т.д.)
```

## 📡 Протокол взаимодействия (Event Bus)

**Никаких прямых ссылок!** Если мусор собран, он не лезет в UI. Он делает так:
`GameEvents.metal_collected.emit(amount)`

**Основные сигналы в `core/game_events.gd`:**
* `garbage_clicked(amount)` — когда игрок тапнул по мусору.
* `resource_changed(type, new_total)` — когда `ResourceManager` обновил данные.
* `build_requested(module_type, position)` — запрос от UI на постройку комнаты.

## 🤝 Правила работы в Git (Чтобы не было конфликтов)

1.  **Правило "Мастер-Сцены":** Старайся не менять `main.tscn`. Если тебе нужно добавить фичу, делай её в отдельной сцене в своей папке, а в `main` мы добавим её один раз как инстанс.
2.  **Атомарные коммиты:** Один коммит — одна фича.
3.  **External Resources:** Все настройки (скорость полета, урон) выносим в `.tres` файлы. Так геймдизайнер может менять баланс, не создавая конфликтов в коде скрипта.
4.  **Уникальные ID:** При создании новых модулей используй строковые ID или Enum, прописанные в `core/constants.gd`.

## 🚀 С чего начать?

1.  Клонируй репозиторий.
2.  Проверь, что в настройках проекта включен **Autoload** для `GameEvents.gd` и `ResourceManager.gd`.
3.  Если ты создаешь новую фичу — создай под неё папку в `entities/` или `ui/`.

---
*Помни: каждый раз, когда ты делаешь `get_node("/root/Main/Some/Very/Long/Path")`, в космосе взрывается одна сверхновая. Используй сигналы!*
